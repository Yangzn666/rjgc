第二章：可行性研究

★可行性研究的任务

可行性研究的任务是确定软件项目是否值得开发，从多方面进行分析论证

主要任务：

  • 技术可行性：从技术角度分析项目是否能实现
  • 经济可行性：分析开发成本与预期收益
  • 操作可行性：评估系统运行的条件和环境
  • 法律可行性：确保项目符合法律法规
★可行性研究的步骤

系统化的可行性研究步骤是确保分析全面性的关键

基本步骤：

  1. 复查系统规模和目标：明确问题边界
  2. 研究现行系统：了解现有流程和问题
  3. 导出高层逻辑模型：抽象出系统核心功能
  4. 推荐建议方案：提出可行的解决方案
  5. 书写文档并提交审查：形成正式的可行性分析报告
★技术可行性分析

技术可行性分析是评估现有技术能否支持项目实现

分析内容：

  • 现有技术能力评估
  • 硬件/软件环境评估
  • 开发人员技能评估
  • 开发时间限制评估
★经济可行性分析

经济可行性分析主要评估项目成本效益

成本构成：

  • 开发成本：人力、设备、软件工具等
  • 运行维护成本：系统维护、人员培训等
  • 预期收益：直接收益和间接收益

分析方法：

  • 成本/效益分析
  • 货币的时间价值
  • 投资回收期
  • 纯收入
★操作可行性分析

操作可行性分析评估系统运行的现实条件

考虑因素：

  • 用户对系统操作的接受程度
  • 组织管理模式的适应性
  • 人员技能水平
  • 工作环境支持
★可行性分析报告

可行性分析报告是可行性研究的最终成果

主要内容：

  • 系统规模和目标说明
  • 技术、经济、操作可行性分析结果
  • 推荐方案：是否继续开发的建议
  • 项目开发计划

结论：项目是否值得投资开发

★数据流图

数据流图(DFD)是需求分析阶段的重要工具

数据流图的四个元素：

  • 外部实体：位于系统之外的信息提供者或消费者，用矩形表示
  • 数据处理：对数据进行加工或变换，用圆角矩形或圆形表示
  • 数据流：数据的流向，用箭头表示
  • 数据存储：数据的静态存储，用开口矩形或双线矩形表示

数据流图的四个符号及其含义：

  • □：外部实体（源点/终点）
  • ○：处理/变换
  • →：数据流
  • 〓：数据存储

作用：

  • 表达数据在系统中流动的情况
  • 描述数据处理过程
  • 作为系统分析和设计的工具
  • 反映系统"做什么"，不反映"怎么做"
★数据字典

数据字典(DD)是结构化分析的重要工具

数据字典的作用：

  • 定义和说明数据流图中的各个成分
  • 对数据流图起到补充作用
  • 保证数据的一致性和完整性
  • 为系统设计提供详细的数据描述

数据字典的内容包括：

  • 数据项：不可再分的数据单位
  • 数据结构：数据项的组合
  • 数据流：数据流名、别名、组成、来源、去向等
  • 数据存储：存储的数据结构
  • 处理过程：处理过程的描述

数据字典中的符号表示：

  • =：等于（定义为）
  • +：与（连接两个数据分量）
  • [ ]：或（方括号内的任选一项）
  • { }：重复（花括号内的数据可重复出现）
  • *
  • ( )：可选项（圆括号内的数据可有可无）

数据字典和数据流图共同构成了系统的逻辑模型

★系统流程图

系统流程图是可行性研究中的重要工具，用于描绘物理系统的传统工具

基本符号：椭圆表示处理、平行四边形表示输入输出等

与数据流图的区别：系统流程图描述系统"怎么做"，数据流图描述系统"做什么"

★成本效益分析方法

成本效益分析是经济可行性分析的核心方法

主要指标：

  • 投资回收期：多长时间能收回投资
  • 纯收入：整个生命周期内的净收益
  • 投资回收率：年平均收益率
