★面向对象方法学概述

面向对象方法学是一种将现实世界中的事物抽象为对象，通过对象之间的交互来解决问题的软件开发方法。

面向对象方法学的特点：

  • 从问题域出发，将客观事物抽象为对象
  • 将数据和对数据的操作封装在一起
  • 强调软件的可重用性、可扩展性和可维护性
  • 更接近人类的思维方式

面向对象方法学的优势：

  • 提高软件的可维护性
  • 增强软件的可重用性
  • 提高开发效率
  • 降低软件开发和维护成本
★面向对象的基本概念

对象(Object)：系统中用来描述客观事物的一个实体，是构成系统的基本单位。

  • 对象由一组属性和对这组属性进行操作的一组服务构成
  • 对象 = 数据 + 算法
  • 属性：描述对象特征的数据项
  • 服务：对数据进行的操作

类(Class)：具有相同属性和服务的一组对象的集合，是一个抽象的概念。

  • 类是对一系列相似对象的抽象描述
  • 对象是类的具体实例
  • 类是对象的模板，对象是类的实例

封装(Encapsulation)：把对象的属性和服务结合成一个独立的系统单位，并尽可能隐蔽对象的内部细节。

  • 信息隐蔽是通过对象的封装性来实现的
  • 对外提供访问接口，内部实现细节对外不可见
  • 提高了系统的安全性和可维护性

继承(Inheritance)：子类可以获得父类的属性和服务，也可以定义自己的属性和方法。

  • 单继承：一个子类只有一个父类
  • 多继承：一个子类可以有多个父类
  • 提高了软件模块的可重用性和可扩充性

多态(Polymorphism)：同一个操作作用于不同类的对象可以有不同的解释，产生不同的执行结果。

  • 允许不同类的对象对同一消息做出响应
  • 灵活性和可扩展性的体现
  • 方法重载和方法重写都是多态的体现
★面向对象分析(OOA)

面向对象分析的任务：运用面向对象方法进行需求分析，建立系统分析模型。

OOA的主要活动：

  1. 识别对象，进行抽象分类
  2. 标识对象间的静态联系
  3. 定义对象的服务，标识对象间的动态联系
  4. 确定对象的操作方法
  5. 确定约束，给出说明
  6. 整理和修改模型

OOA模型：

  • 对象模型：描述系统的静态结构
  • 动态模型：描述系统的交互序列
  • 功能模型：描述系统的数据变换功能

OOA的特点：

  • 从现实世界中的实体出发认识问题
  • 建立的分析模型易于理解和交流
  • 分析模型对需求的变化具有较强的适应性
★面向对象设计(OOD)

面向对象设计的任务：在OOA模型的基础上，对分析模型进行补充和完善，形成设计模型。

OOD的主要内容：

  • 问题域部分设计
  • 人机交互部分设计
  • 任务管理部分设计
  • 数据管理部分设计

OOD设计准则：

  • 抽象：抽取事物的本质特征
  • 信息隐蔽：对象的实现细节对外部是隐蔽的
  • 功能独立：对象的内部关联要尽量紧密
  • 一致性：设计应保持内在的一致性
  • 完备性：设计应能完全表达问题空间的需求

OOD设计风格：

  • 对象类粒度适中
  • 继承层次适当
  • 消息关联尽量简单
  • 对象间耦合程度要低
★面向对象编程(OOP)

面向对象编程的特点：

  • 封装性：将数据和操作数据的方法绑定在一起
  • 继承性：子类可以继承父类的属性和方法
  • 多态性：同一个接口可以有不同的实现

常用面向对象编程语言：

  • C++：在C语言基础上增加了面向对象特性
  • Java：纯面向对象编程语言，跨平台性强
  • C#：微软开发的面向对象语言
  • Python：支持多种编程范式，包括面向对象

OOP的优点：

  • 可重用性高
  • 可维护性好
  • 可扩展性强
  • 代码可读性好
★UML统一建模语言

UML概述：Unified Modeling Language，是一种通用的可视化建模语言，用于软件密集型系统的分析、设计和构造。

UML图的分类：

  • 结构图：描述系统的静态结构
    • 类图：描述类、接口以及它们之间的关系
    • 对象图：表示类的对象实例
    • 组件图：描述组件及其依赖关系
    • 部署图：描述系统中硬件和软件的物理配置
  • 行为图：描述系统的动态行为
    • 用例图：描述系统功能及参与者与系统的关系
    • 活动图：描述业务流程或操作的工作流程
    • 状态图：描述对象的状态转换
    • 序列图：描述对象间的交互序列

UML的优势：

  • 标准化的建模语言
  • 可视化程度高
  • 与具体实现无关
  • 支持面向对象的分析、设计和实现
★面向对象建模技术3种模型之间的三种关系

面向对象建模技术中建立的三种模型（对象模型、动态模型、功能模型）之间存在密切的关系。

三种模型的定义：

  • 对象模型：描述系统的静态结构，即系统中各类以及它们之间的关系
  • 动态模型：描述系统中对象之间的交互序列，即系统的行为
  • 功能模型：描述系统中数据的变换功能，即系统的功能

三种模型之间的关系：

  • 对象模型是最基础的模型：定义了系统中的对象和它们之间的关系，是其他两个模型的基础
  • 动态模型描述对象的行为：定义了对象在系统生命期中行为的改变，描述了对象之间的交互
  • 功能模型描述对象的功能：描述了数据在系统中的流动和变换，即对象所提供的功能
  • 三个模型相互补充：从不同角度描述系统，共同构成完整的系统模型

建模顺序：通常先建立对象模型，再建立动态模型，最后建立功能模型。

★软件项目管理概述

软件项目管理的定义：软件项目管理是为了使软件项目能够按照预定的成本、进度、质量顺利完成，而对成本、人员、进度、质量、风险等进行分析和管理的活动。

软件项目的特点：

  • 智力密集型：高度依赖人的创造性思维
  • 劳动密集型：需要大量人力投入
  • 产品无形性：软件是逻辑产品，不可见
  • 开发复杂性：涉及多学科知识的综合应用
  • 过程多变性：需求变更频繁

软件项目管理的重要性：

  • 提高项目成功率
  • 控制成本和进度
  • 保证产品质量
  • 降低项目风险
  • 提高客户满意度
★软件项目组织管理

项目组织结构：

  • 民主制程序员组：
    • 所有成员地位平等，共同决策
    • 适合规模较小、技术水平较高的团队
    • 优点：沟通效率高，成员积极性强
    • 缺点：缺乏统一指挥，决策效率低
  • 主程序员组：
    • 由主程序员、后备程序员、编程秘书组成
    • 主程序员负责主要设计和编码工作
    • 适合技术复杂、质量要求高的项目
  • 层次式程序员组：
    • 设立组长，组员向组长汇报
    • 适合规模较大、层次分明的项目
    • 优点：权责明确，管理有序
    • 缺点：沟通效率可能受影响

团队建设：

  • 选择合适的团队成员
  • 建立良好的团队文化
  • 加强团队沟通与协作
  • 设定明确的团队目标
★软件项目进度管理

进度计划制定：

  • 分解工作任务（WBS - 工作分解结构）
  • 估算工作量和持续时间
  • 确定任务间的依赖关系
  • 分配资源和人员

进度管理工具：

  • 甘特图（Gantt Chart）：
    • 横轴表示时间，纵轴表示任务
    • 用条状图显示任务的起止时间和持续时间
    • 直观反映项目进度状况
  • 网络图（PERT/CPM）：
    • 节点表示事件，箭线表示活动
    • 能够识别关键路径
    • 有助于优化项目时间安排
  • 里程碑（Milestone）：
    • 标志项目重要阶段的完成
    • 便于项目监控和控制
    • 提供项目进展的可见标志
★软件项目成本管理

软件成本构成：

  • 人员费用：占总成本的60%-80%
  • 软硬件设备费用
  • 培训和支持费用
  • 管理费用

成本估算方法：

  • 专家估算法：依靠专家经验进行估算
  • 类比估算法：参考类似项目的成本数据
  • 算法模型法：使用数学模型进行估算
  • 自下而上法：逐项估算后汇总

常用的估算模型：

  • COCOMO模型：构造性成本模型
    • 基本COCOMO：静态单变量模型
    • 中级COCOMO：考虑多个成本驱动因素
    • 详细COCOMO：按软件生命周期各阶段分别估算
  • Putnam模型：动态多变量模型，基于工作量分布曲线
★面向对象方法学的优点与缺点

面向对象方法学作为一种重要的软件开发方法，具有明显的优点和一定的局限性。

面向对象方法学的优点：

  • 可重用性高：通过继承和封装，可以在新项目中重用已有代码
  • 可维护性好：封装性使得修改局部不会影响整体系统
  • 可扩展性强：通过继承和多态，可以方便地扩展系统功能
  • 开发效率高：利用现成的类库和框架，加速开发过程
  • 更接近人类思维方式：符合人们对客观世界的认知习惯
  • 信息隐蔽性好：通过封装隐藏实现细节，只暴露必要的接口

面向对象方法学的缺点：

  • 学习曲线陡峭：需要掌握面向对象的概念和设计原则
  • 对开发人员要求较高：需要具备良好的抽象思维能力
  • 前期设计投入较大：需要仔细设计类结构和关系
  • 对于某些问题域不太适用：如系统级编程、算法密集型应用

与传统方法学的比较：

  • 传统方法以功能分解为核心，面向对象方法以对象为核心
  • 传统方法自顶向下设计，面向对象方法更注重对象间的关系
  • 传统方法使用结构化分析和设计，面向对象方法使用面向对象分析和设计
★面向对象方法学的应用与发展

面向对象方法学在现代软件开发中有着广泛的应用，并且不断发展演进。

面向对象方法学的应用领域：

  • 企业级应用开发：如ERP、CRM等大型管理系统
  • Web应用程序开发：现代Web框架多采用面向对象设计
  • 移动应用开发：Android、iOS平台开发都基于面向对象
  • 桌面应用程序：GUI应用多采用面向对象的事件驱动模型
  • 游戏开发：游戏中角色、道具等天然适合面向对象建模

面向对象方法学的发展趋势：

  • 组件化开发：发展为基于组件的软件开发（CBD）
  • 模型驱动开发：发展为模型驱动架构（MDA）
  • 领域驱动设计：更注重业务领域的建模
  • 函数式编程融合：与函数式编程理念相结合

面向对象与其他技术的结合：

  • 与设计模式结合，提高代码质量
  • 与UML建模语言结合，提高分析设计效率
  • 与敏捷开发方法结合，提高开发效率
  • 与微服务架构结合，构建分布式系统
